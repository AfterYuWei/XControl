# Skill：Hairline Shadow 高级质感卡片阴影实现

#### 技能定位

针对 Card、Modal、Popover 等浮层组件，用阴影体系替代独立边框，解决边缘发灰生硬问题，提升界面精致度与层级统一感。

#### 适用场景

所有需要表达「悬浮层级」的 Surface 类组件：卡片、弹窗、下拉菜单、浮层提示、侧边抽屉、命令面板、悬浮工具栏等。

#### 核心问题

传统 `border + box-shadow` 组合中，固定几何属性的边框，与渐变过渡的阴影视觉逻辑冲突，在浅色背景下易在边缘形成生硬的灰带，破坏界面高级感。

#### 实现原理

将组件的边缘轮廓纳入阴影系统，用**零位移、零模糊的阴影扩散层（hairline）** 替代独立 border，让边缘从阴影中自然过渡；同时将阴影拆分为三层，分别承担不同视觉作用，层级表达更精准。

#### 标准实现步骤

1. 移除组件的独立 `border` 属性

2. 按顺序叠加三层 `box-shadow`（绘制顺序从前到后，视觉上从内到外）：

    - 第一层「边缘环」：无位移、无模糊、1px 扩散，低透明度深色，用于柔和界定元素边界

    - 第二层「接触阴影」：小位移、小模糊，模拟元素贴近背景的自然暗部

    - 第三层「环境阴影」：大位移、大模糊，营造整体悬浮的空间层级

基础代码示例：

```css
.surface-card {
  background: #ffffff;
  border-radius: 12px;
  box-shadow:
    0 0 0 1px rgba(0, 0, 0, 0.08),  /* hairline 边缘环 */
    0 1px 2px rgba(0, 0, 0, 0.05),  /* 接触阴影 */
    0 4px 12px rgba(0, 0, 0, 0.06); /* 环境阴影 */
}
```

#### 参数调优规则

- 极浅背景：将边缘环透明度下调至 0\.06，弱化边缘存在感

- 小型紧凑组件：收窄环境阴影的模糊半径与位移值，避免阴影过重

- 高密度工具界面：边缘环透明度不超过 0\.12，防止重新出现生硬边框感

- 高分屏适配：可使用 0\.5px 扩散的边缘环；生产环境优先选择「1px \+ 低透明度」，跨浏览器兼容性更稳定

#### 进阶优化：内侧顶部高光

针对白色卡片顶部被阴影压暗的问题，可新增一层极淡的内阴影补充高光，提升通透感：

```css
.surface-card {
  box-shadow:
    inset 0 1px 0 rgba(255, 255, 255, 0.9), /* 顶部内高光，需极淡 */
    0 0 0 1px rgba(0, 0, 0, 0.08),
    0 1px 2px rgba(0, 0, 0, 0.05),
    0 4px 12px rgba(0, 0, 0, 0.06);
}
```

#### 禁用场景（仍需使用 border /outline）

- 输入框、文本域、下拉选择等表单控件的结构边界

- focus、error、selected 等状态标识的边框

- 表格线、布局分隔线等强结构属性的线条

- 需要与布局尺寸强绑定的分割线
